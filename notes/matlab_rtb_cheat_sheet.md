MATLAB 与 Robotics Toolbox (RTB) 核心知识速查手册



本手册将我们在跑通“重载机械臂仿真底盘”过程中遇到的所有基础概念，进行了模块化分类。方便在未来忘记语法时，随时查阅。



🛠️ 第一类：Robotics Toolbox (RTB) 专属核心函数



注意：以下函数均不是 MATLAB 自带的，必须在成功配置 RTB 工具箱后才能使用。



1\. SerialLink (串联机械臂组装)



作用：机械臂的“组装流水线”。



精讲：负责将预先定义好的一根根独立的连杆（Link），按照先后顺序首尾相连，打包成一个完整的机械臂对象（通常命名为 robot）。



用法：robot = SerialLink(\[L1 L2 L3 L4], 'name', 'Heavy\_Arm');



2\. fkine (Forward Kinematics - 正运动学)



作用：已知“关节角度”，求末端夹爪的“空间位置”。



精讲：你输入一组电机的转动角度，工具箱底层会自动进行极其复杂的 $4 \\times 4$ 齐次变换矩阵相乘，最终输出末端在三维空间中的坐标 $(X,Y,Z)$ 和姿态。



3\. ikine (Inverse Kinematics - 逆运动学)



作用：已知“目标空间位置”，求“电机该转多少度”。



精讲：这是未来接入 AI 大模型的最核心接口（翻译官）。大模型给出 $(X,Y,Z)$ 坐标（通过 transl 转换后喂给此函数），它负责算出对应的关节角度 $q$。



⚠️ 4轴机械臂专属避坑：对于胡师兄的 4 轴机械臂，因为做不到空间全姿态，调用时必须加掩码：q = robot.ikine(T, 'mask', \[1 1 1 0 0 0])。（前三个 1 代表必须满足 XYZ 坐标，后三个 0 代表忽略姿态要求）。



4\. transl (空间坐标转换器)



作用：纯粹的数学转换工具。



精讲：把人类（或 AI）给出的 (X, Y, Z) 纯坐标数字，瞬间转换成机器人学算法必须要用的 $4 \\times 4$ 标准齐次变换矩阵。



💻 第二类：MATLAB 基础语法 (对比 Python)



1\. 代码注释 (% 与 %{ %})



单行注释：MATLAB 使用 %（完全等同于 Python 里的 #）。



多行注释：MATLAB 使用 %{ 开始，并以 %} 结束（功能等同于 Python 里的 '''）。



2\. 终端打印输出 (disp)



作用：向命令行窗口输出文字或变量。



精讲：disp(变量) 是最简单的打印，等同于 Python 里的 print(变量)。



进阶用法：如果需要把文字和数字拼接在一起打印，MATLAB 更常用的是 fprintf('结果是: %f\\n', 变量)，这类似于 Python 里的 print(f"结果是: {变量}")。



📊 第三类：变量定义与图形可视化



1\. q 变量 (广义坐标/关节角度)



概念：在机器人学里，字母 $q$ 通常约定俗成用来代表“关节角度”。



用法：例如 q\_test = \[0, pi/4, pi/3, -pi/4]。这是一个包含 4 个数字的数组（列表），明确定义了 4 个电机在此刻分别应该转动多少弧度。



2\. figure (激活画板)



作用：创建或激活一个独立的图形窗口。



精讲：figure(1) 并不是给图表命名！它的实际指令是：“电脑，请给我拿一张编号为 1 的空白画板出来”。紧接着运行的 .plot() 函数，就会自动把图画在这张 1 号画板上。



3\. title (图表命名)



作用：真正给图形窗口加上标题的函数。



用法：紧跟在画图代码之后使用。例如：title('正运动学演示：已知角度求位置');，这行字会显示在画板的正上方。
